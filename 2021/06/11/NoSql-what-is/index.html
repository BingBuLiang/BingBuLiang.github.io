2021-06-11
NoSQL

NoSql简介

简介

  • NoSQL ：Not Only SQL
  • 泛指非关系型数据库的
  • 随着==Web2.0==互联网的诞生！传统的关系型数据库很难对付Web2.0时代！
  • 尤其是超大规模的高并发的社区
    • 暴露出来很多难以克服的问题
    • NoSQL在当今大数据环境下发展十分迅速
    • Redis是发展最快的,而且是我们当下必须掌握的一个技术！

数据类型

  • 很多数据类型的存储不需要一个**==固定的格式==**！不需要多余的操作就可以横向扩展的！

    • 用户的个人信息
    • 社交网络
    • 地理位置

  • 可以以Map<String,Object>使用键值对的方式来控制

特点

  • 方便扩展

    • 数据之间没有关系，就可以很好的扩展

  • 大数据量，高性能

    • Redis的性能

      • 写：8万次/s
      • 读：11万次/s

    • NoSQL缓存记录级，是一种细颗粒的缓存，性能会比较高

  • 数据类型是多样型的

    • 不需要事先设计数据库！
    • 随取随用
    • 如果是数据量十分大的表，很多人就无法设计了

传统的 RDBMS和NoSQL的区别

RDBMS

  • 结构化组织
  • SQL
  • 数据和关系都存储在单独的表中
  • 数据定义语言，数据操作语言
  • 严格的一致性
  • 基础的事务
  • …..

NoSQL

  • 不仅仅是数据

  • 没有固定的查询语言

  • 存储方式

    • 键值对存储
    • 类存储
    • 列存储
    • 图形数据库：社交关系

  • 最终一致性

  • CAP定理和BASE

    • 异地多活

  • 保证高性能，高可用，高扩展

  • ……

扩展

3V+3高问题

  • 大数据时代的3V:主要是描述问题
    • 海量
    • 多样
    • 实时
  • 大数据时代的3高:主要是对程序的要求
    • 高并发
    • 高可扩
      • 随时水平拆分，机器不够了，可以通过加机器解决
    • 高性能

真正在公司中的实践：NoSQL+RDBMS一起使用的才是最强大的
